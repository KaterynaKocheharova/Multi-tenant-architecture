# ADR-005: Frontend Framework

## Context

Платформа є закритою multi-tenant SPA (school management system) без вимог до SEO. Основні потреби:

- Динамічний UI
- In-memory зберігання `accessToken` для зменшення ризику XSS-крадіжки (ADR-003)
- Автоматичний silent refresh через `POST /auth/refresh` (refresh token у HttpOnly cookie)
- Захист маршрутів на рівні компонентів (`ProtectedRoute`)
- Команда має наявну експертизу в React-екосистемі

Серверний рендеринг (SSR) та SEO не є вимогами: платформа доступна лише авторизованим користувачам і не індексується пошуковими системами.

## Consequences

### Positive

- **Наявна експертиза** — команда знає React, що зменшує ризик затримок через навчання
- **Простий деплой** — Vercel з нативною підтримкою Vite SPA

### Negative

- **Відсутність SSR і React Server Components** — Vite SPA є повністю клієнтським: весь JS bundle завантажується браузером, RSC недоступні (React Server Components вимагають серверного runtime — Next.js/Remix/тощо, якого у статичному SPA немає). Некритично для закритої платформи, але відчутно на повільних з'єднаннях

## Alternatives Considered

### Next.js (SSR/SSG)

**Розглядалось:** React-фреймворк із вбудованим SSR, file-based routing, API routes, оптимізацією зображень.

**Відхилено:**

- SSR не дає цінності для закритої платформи — всі сторінки потребують автентифікації, пошукова індексація не потрібна
- Складніше зберігати `accessToken` in-memory в SSR-контексті (server-side рендеринг не має доступу до браузерного Zustand-стору)
- Вища складність деплою (serverless functions vs статичний SPA)
- Надлишкова складність для поточних вимог

### Vue.js (Vite SPA)

**Розглядалось:** Легший фреймворк із реактивністю з коробки, Pinia як state manager.

**Відхилено:** Команда не має Vue-експертизи. Екосистема менша ніж у React — більше часу на пошук рішень для специфічних задач (tenant-scoped кешування, magic link flow тощо). Ризик затримок через навчання неприйнятний в умовах часових обмежень.

### Angular

**Розглядалось:** Повноцінний opinionated фреймворк із DI, RxJS, вбудованим роутингом і HTTP-клієнтом.

**Відхилено:** Значно більший бойлерплейт і крутіша крива навчання. Відсутня існуюча експертиза в команді.
