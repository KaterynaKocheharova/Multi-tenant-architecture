# BOUNDED CONTEXT

## Контекст управління тенантами (рівень платформи)

### Сутності

- Tenant

### Поведінка

- реєстрація школи
- активація / призупинення

### Відповідальність

- Відповідає за lifecycle тенанта
- Відповідає за глобальний реєстр тенантів

## Контекст ідентифікації

### Сутності

- User
- TeacherDetails
- StudentDetails
- Membership
- MembershipRole

### Поведінка

- автентифікація юзера
- застосування меж доступу на рівні школи
- перегляд та редагування даних юзера
- визначення приналежності до того чи іншого тенанта

### Відповідальність

- Відповідає за автентифікацію та авторизацію
- Відповідає за зв'язок юзера з тенантом

## Контекст управління подіями

### Сутності

- Event (вебінар, концерт, змагання)
- EventParticipation
- CompetitionParticipation

### Поведінка

- створення події
- реєстрація
- додавання посилання на відео виступу/вебінару
- участь
- відмітка відвідуваності
- призначення місця
- надсилання сертифіката

### Відповідальність

- Відповідає за lifecycle відвідуваності
- Відповідає за стан участі у події

## Контекст планування уроків

### Сутності

- LessonPlan
- LessonPlanAssignment

### Поведінка

- створення плану
- створення шаблону
- створення з шаблону
- призначення плану учню

### Відповідальність

- Відповідає за логіку планування уроків
- Відповідає за призначення планів

## Контекст звітності (Read Model)

### Основна концепція

- Звіт **обчислюється та зберігається**

### Вхідні дані

- EventParticipation (Контекст подій)
- LessonPlanAssignment (Контекст уроків)

### Поведінка

- агрегація даних
- обчислення результатів
- генерація звітів
- поєднання подій
- облік проведених уроків
- збереження згенерованих звітів

### Відповідальність

- Відповідає за логіку агрегації та збережені звіти
